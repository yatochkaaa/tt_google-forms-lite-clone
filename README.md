# Google Forms Lite Clone

A monorepo implementation of a Google Forms-like application built as a take-home test task.

## Stack

| Layer | Technology |
|---|---|
| Frontend | React 19, TypeScript, Vite |
| State / Data fetching | Redux Toolkit + RTK Query |
| GraphQL client | `graphql-request` + `@rtk-query/graphql-request-base-query` |
| Type generation | `graphql-codegen` → typed RTK Query hooks |
| Forms | `react-hook-form` |
| Styling | Tailwind CSS v4 |
| Routing | React Router v7 |
| Backend | NestJS, GraphQL (code-first, Apollo) |
| Data store | In-memory `Map` (no database) |
| Package manager | pnpm workspaces |

## Project Structure

```
.
├── client/          # React application
│   └── src/
│       ├── api/     # .graphql operation files + generated RTK hooks
│       ├── components/
│       ├── hooks/   # Business logic (useFormBuilder, useFormFill, …)
│       ├── pages/
│       └── store/   # RTK store + base API
└── server/          # NestJS GraphQL API
    └── src/
        ├── forms/   # Forms resolver, service, models, DTOs
        └── responses/ # Responses resolver, service, models, DTOs
```

## Getting Started

### Prerequisites

- Node.js 18+
- pnpm 9+ (`npm install -g pnpm`)

### Install dependencies

```bash
pnpm install
```

### Approve build scripts (first time only)

```bash
pnpm approve-builds
```

### Run in development

```bash
pnpm dev
```

This starts both the client and server concurrently:

- Client: [http://localhost:5173](http://localhost:5173)
- Server (GraphQL): [http://localhost:3000/graphql](http://localhost:3000/graphql)

## Architecture Notes

### Type sharing via codegen

GraphQL schema is generated by NestJS at startup (`schema.gql`). The codegen tool reads it together with `.graphql` operation files in `client/src/api/` and generates fully-typed RTK Query hooks in `client/src/api/generated.ts`. No manual type duplication between client and server.

### RTK Query cache invalidation

Responses are tagged with `"Response"` via `api.enhanceEndpoints()`. Submitting a response via `SubmitResponse` mutation automatically invalidates the `GetResponses` cache, so the Responses page updates reactively without a manual refresh.

### Business logic in hooks

Components contain only JSX. All data fetching, mutations, form state, and derived logic live in dedicated hooks (`useFormBuilder`, `useFormFill`, `useFormResponses`, `useForms`).
